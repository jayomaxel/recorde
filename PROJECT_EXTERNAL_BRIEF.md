# Ethereal Notes 对外汇报版（Product Brief）

版本日期：2026-03-03
适用对象：产品、业务、项目管理、跨团队同步

## 1. 项目一句话
Ethereal Notes 是一款本地优先（Local-first）的 AI 思绪记录应用，帮助用户低门槛记录情绪轨迹，并从历史内容中获得可执行的反思建议。

## 2. 当前交付价值
- 记录效率：用户可快速写下当下思绪，支持编辑、收藏、搜索、随机回看。
- AI 增益：可对单条思绪自动生成情绪（mood）与标签（tags），并汇总趋势。
- 洞察能力：基于历史记录生成三段式深度分析（现状、模式、建议）。
- 数据可控：默认仅存本地浏览器，不依赖平台账号系统。

## 3. 已完成能力（可上线范围）
### 3.1 用户流程
- 首次引导：昵称、ID、邮箱、4 位 PIN、风险确认。
- 隐私锁屏：支持 PIN 解锁与手动输入。
- 忘记密码闭环：可“重置应用”清空本地数据并重新初始化。

### 3.2 AI 能力
- 支持 Gemini Native 与 OpenAI 兼容协议。
- 情绪分析支持人格风格配置：`philosophical / poetic / concise`。
- 深度分析支持流式输出，交互反馈及时。

### 3.3 工程质量
- 开发端口统一为 `3002`。
- 已修复 `favicon` 404。
- 已补可访问性基础项（icon 按钮可识别文本、表单字段命名）。
- 已补兼容性基础项（`text-size-adjust`、Safari `-webkit-backdrop-filter`）。
- 已补 dev/preview 响应头基础策略（`nosniff`、`Cache-Control` 优先等）。

## 4. 产品边界（必须对齐）
- 本版本不提供云同步与多端一致性。
- 本版本不提供账号找回机制。
- 忘记 PIN 后历史加密数据不可恢复，只能重置应用。
- 启用 AI 后，内容会发送到用户配置的第三方模型服务。

## 5. 风险与现状说明（对外口径）
- 已做本地轻量安全增强，但当前不是企业级安全方案。
- 当前密码学实现以“可用和迭代速度优先”，后续需升级标准密码学能力。
- 深度分析目前按“最多 30 条记录”控制上下文，尚未引入 token 预算器。
- 密码变更涉及本地重加密，暂在主线程执行，超大数据量下有卡顿风险。

## 6. 下一阶段计划（建议）
1. 安全升级：AES-GCM + PBKDF2/Argon2（替换轻量方案）。
2. 体验升级：密码变更加密迁移到 Web Worker，避免主线程卡顿。
3. 成本与稳定性：引入 token 估算与截断策略，控制上下文和费用。
4. 可运维性：补齐 AI 超时、重试、断路和结构化错误码。

## 7. 对业务的直接意义
- 可作为“个人成长记录 + AI 反思”场景的 MVP 验证样板。
- 可沉淀真实用户行为数据，为后续云同步、订阅策略、B2B 私有化版本提供决策依据。
- 在不引入后端复杂度的前提下，快速验证“记录 -> 洞察 -> 留存”的核心价值链。

## 8. 对外沟通建议（统一话术）
- 我们是“本地优先 + AI 洞察”的效率工具，而非医疗诊断产品。
- 我们强调用户数据控制权与私密性，但不宣称企业级加密。
- 当前版本重点是验证用户价值闭环，下一阶段再做安全和规模化增强。
