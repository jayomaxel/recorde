# Ethereal Notes 深度说明（答辩版）

## 1. 文档目标
本文件用于应对三类高频盘问：
- Dev：架构、数据一致性、安全实现细节。
- PM：用户预期、风险提示、边界定义。
- Algo：Prompt 策略、上下文控制、模型兼容性。

所有结论均基于当前仓库实现，不做“未来能力”冒充。

## 2. 一句话定位
一个 Local-first 的思绪记录与 AI 洞察应用，核心闭环是：
记录 -> 标注情绪/标签 -> 聚合趋势 -> 生成深度分析。

## 3. 关键架构事实
- 前端 SPA，无自有后端。
- 持久化全部在 `localStorage`。
- AI 请求直连用户配置的第三方服务（Gemini Native 或 OpenAI 兼容）。
- 锁屏与会话解锁逻辑在前端完成。

## 4. Dev 盘问（工程与安全）
### Q1：改密码后会发生什么？
结论：支持改密码，且会在保存设置时重写本地加密数据。

当前链路：
1. 用户在 Settings 输入旧 PIN 与新 PIN。
2. 前端先验证旧 PIN（哈希比对）。
3. 保存设置时，读取当前思绪数据并按新 PIN 重新加密后写回。

边界与风险：
- 该重加密在主线程执行，数据量大时会有卡顿风险。
- 当前未引入 Web Worker。

### Q2：是否还在存明文密码？
结论：不再存明文密码。

当前策略：
- 设置中持久化字段是 `passwordHash`。
- 运行时明文 PIN 仅保存在会话内存变量。
- 退出登录会清理会话敏感缓存。

### Q3：忘记密码怎么办？
结论：无找回机制，只有“重置应用”。

实现现状：
- 锁屏页提供“重置应用”按钮。
- 操作结果：清空本地数据并刷新页面。

产品语义：
- 这是“安全优先”的取舍，不是 bug。
- 必须配合强提示文案进行预期管理。

### Q4：加密是否达到企业级？
结论：没有。

当前是轻量方案：
- 哈希：FNV1a 变体（用于本地校验）。
- 加密：XOR + salt + 前缀封装。

不应对外表述为：
- 零知识密码学方案
- 企业级强加密存储

## 5. PM 盘问（体验与预期）
### Q1：用户会不会误以为可找回密码？
风险存在，但已加防御：
- Onboarding 红字警告 + 勾选确认。
- 修改密码弹窗红字警告 + 勾选确认。
- 锁屏页提供“忘记密码 -> 重置应用”闭环。

仍建议：
- 在产品对外说明中写明“遗忘 PIN 不可恢复”。
- 在导出能力旁提示“建议定期导出备份”。

### Q2：为什么做 Local-first？
收益：
- 隐私控制更直接。
- 实现复杂度低，迭代快。

代价：
- 天然无跨端同步。
- 数据容灾能力依赖用户主动导出。

## 6. Algo 盘问（AI 实现）
### Q1：`aiPersonality` 是不是装饰字段？
结论：不是。已接入 Prompt 策略。

作用范围：
- `analyzeThought`：影响情绪分析提示词风格。
- `streamDeepInsight`：影响深度分析输出风格。

模式：
- `philosophical`
- `poetic`
- `concise`

### Q2：深度分析会不会上下文溢出？
当前控制：
- 只取最近 30 条记录。

已知不足：
- 没有 token 估算器。
- 没有按 token budget 截断。
- 长文本场景可能导致费用上升或响应变慢。

结论：
- 当前是“条数限流”，不是“token 级控制”。
- 若要面向重度用户，需补 token 预算与截断策略。

### Q3：Gemini / OpenAI 兼容到什么程度？
已支持：
- 连通性测试
- 单条情绪分析
- 深度分析流式输出（Gemini Native 与 OpenAI 兼容路径均有实现）

## 7. 审计项闭环（前端）
已处理：
- `favicon.ico` 404：已补 `favicon.svg` + `favicon.ico`。
- 可访问性：icon-only 按钮补 `aria-label/title`。
- 表单规范：关键输入项补 `id/name`。
- 兼容性：`text-size-adjust` 与 Safari `-webkit-backdrop-filter`。
- Dev/Preview 响应头：`nosniff`、`Cache-Control`、移除 `Expires/X-XSS-Protection`、内容类型归一化。

注意：
- 响应头策略在 Vite dev/preview 生效。
- 生产环境最终以部署平台（Nginx/CDN/网关）配置为准。

## 8. 可直接对外的“真实口径”
- 我们是“本地优先 + 轻量安全”方案，不是企业级安全产品。
- 忘记 PIN 不可恢复，提供重置应用作为唯一兜底路径。
- AI 已支持人格风格，但上下文控制目前按条数，不是按 token。
- 当前版本已完成基础可访问性与兼容性修复，可进入持续打磨阶段。

## 9. 下一步（按优先级）
1. 密码学升级：AES-GCM + PBKDF2/Argon2。
2. 重加密流程迁移到 Web Worker。
3. 深度分析加入 token 预算与截断。
4. AI 错误体系升级：超时/重试/断路/可观测错误码。
